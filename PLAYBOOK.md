# 進場訊號指標研究報告
**生成日期:** 2026-06-12  
**研究方向:** 台股動能/突破策略的進場訊號改善

---

## 現行策略進場條件
- MACD 四箭頭 ≥ 3（DIF>0、DIF 上升、Signal 上升、Histogram 上升）
- ADX > 20
- Williams %R < -20
- 突破 20 日高點

---

## 1. RSI 隱性背離（趨勢延續過濾）

**計算方式：** RSI(14) 日線。隱性多頭背離 = 股價創更高低點，而 RSI 卻創更低低點（同一回溯窗口 10~20 根 K 棒）。

**整合方式：** 作為 MACD 箭頭的輔助正向過濾器。進場條件需滿足：存在隱性背離 **或** RSI(14) > 50（確認動能方向）。背離訊號較強，RSI > 50 為較寬鬆門檻。

**假突破減少：** 鎖定趨勢中的回調再進場，而非追高。隱性背離代表回調動能減弱。

**台股注意事項：** 薄薄的股票因盤中波動劇烈，容易產生假的波段低點。要求低點至少維持 3 根 K 棒後才確認。避免用於 NT$20 以下或日均成交金額 < NT$50M 的股票。

---

## 2. BB/KC 擠壓（波動壓縮突破確認）

**計算方式：** BB = 20日SMA ± 2標準差；KC = 20日EMA ± 1.5×ATR(10)。擠壓成立 = BB 上軌 < KC 上軌 **且** BB 下軌 > KC 下軌。釋放 = BB 重新突破 KC 外側。

**整合方式：** 要求在 20 日高點突破同一根 K 棒（或前後 2 根內）發生擠壓釋放，確認突破是從壓縮帶逃逸，而非高波動時的隨機突破。

**假突破減少：** 實務估計可過濾掉 30~40% 的假突破。搭配現有 ADX > 20 效果更佳。

**台股注意事項：** 台灣 ±10% 漲跌停板可能造成人工擠壓後的暴力釋放。確認訊號須以**收盤價**為準，不可用盤中。若突破 K 棒振幅 > 2×ATR(10)，忽略（可能是跳空/漲停掛牌）。

---

## 3. ATR 擴張確認（最易實作）

**計算方式：** ATR(10) 突破當根 K 棒 ÷ ATR(10) 的 20 日滾動平均 = ATR Ratio

**整合方式：** 要求進場當根 ATR Ratio ≥ 1.3（突破 K 棒振幅比近期基準寬 30%）。

**程式碼（2 行）：**
```python
atr14 = true_range.rolling(14).mean()
atr_ratio = atr14 / atr14.rolling(20).mean()
# 進場條件加：atr_ratio[i] >= 1.3
```

**台股注意事項：** 漲停板股票 ATR Ratio 極高但非真實突破。將 ATR Ratio 上限設為 3.0 以排除漲停事件。

---

## 4. Chaikin Money Flow（CMF）— 法人蓄積門檻

**計算方式：** CMF(20) = Σ[((收-低)-(高-收))/(高-低)×量, 20根] / Σ[量, 20根]，範圍 -1 到 +1

**整合方式：** 要求進場時 CMF > +0.05（避開零線附近的雜訊區）。過濾掉「股價上漲但實際是賣壓」的假突破，台灣散戶主導的中小型股尤其常見此現象。

**程式碼（~10 行）：**
```python
mf_mult = ((close - low) - (high - close)) / (high - low).replace(0, np.nan)
mf_vol  = mf_mult * volume
cmf20   = mf_vol.rolling(20).sum() / volume.rolling(20).sum()
# 進場條件加：cmf20[i] > 0.05
```

**台股注意事項：** TWSE 外資買賣超為當日公布的較佳替代資料。自營商造市（權證發行商）可能扭曲成交量分布，需注意。

---

## 5. 強收盤 K 棒過濾（最簡單）

**計算方式：** 突破當根 K 棒需滿足：(收盤-開盤)/(最高-最低) ≥ 0.6 **且** 收盤在 K 棒振幅上 30% 區間內。

**整合方式：** 單根 K 棒過濾，直接加入進場條件。

**程式碼（1 行）：**
```python
strong_close = ((close - open_) / (high - low + 1e-9) >= 0.6) & (close >= low + 0.7*(high - low))
```

**台股注意事項：** 排除漲停（high==close 且 close == 昨收×1.10）的情況，漲停後常見跳空回落。

## 現行機制
- **BULL:** 0050 MACD 四箭頭 ≥ 3 **且** ADX > 20
- **ALERT:** 四箭頭 1~2 或 BULL 條件未完全滿足
- **BEAR:** 四箭頭 = 0
- **CRASH:** BEAR + 0050 三日內跌 ≥ 3% → 清倉

**已知問題：**
- 2016 年任何版本都虧 -17%
- ADX > 20 過濾器使 2004/2005 早期多頭錯失進場機會

---

## 1. 價格 MA 機制（最簡單 — 1 行）

**計算：** 0050 收盤 vs SMA(120)，加上 SMA(120) 斜率（SMA120 > 前20日 SMA120）

- **BULL：** close > SMA120 **且** SMA120 斜率為正
- **BEAR：** close < SMA120

**為什麼幫助 2016：** 2016 年台灣受人民幣貶值恐慌、中國放緩拖累，1~2 月緩慢下跌。MACD 箭頭在盤整市場可能模糊不清長達數週；價格/MA 機制早在 1 月就會轉 BEAR 並維持。ADX 在橫盤整理時保持低水位，恰好是當前過濾器失效的場景。

**預期改善：** 2016 回撤有望減少 5~10%，對趨勢年影響微小。

---

## 2. 0050 均線交叉機制（簡單 — 5 行）

**計算：** EMA(20) vs EMA(60) 交叉，而非 MACD 箭頭

- **BULL：** EMA20 > EMA60 **且** EMA60 上升
- **BEAR：** EMA20 < EMA60

**vs MACD+ADX：** MACD 衡量動能背離；MA 交叉衡量趨勢對齊。MA 交叉較慢觸發（橫盤年較少假訊號），但在早期多頭行情確認較快（修正 2004/2005 被 ADX 擋住的問題）。

**程式碼：**
```python
ema20 = close.ewm(span=20).mean()
ema60 = close.ewm(span=60).mean()
bull_signal = (ema20 > ema60) & (ema60 > ema60.shift(5))
```

---

## 3. 波動率機制門檻（中等 — 10 行）⭐ 最高 ROI

台灣沒有公開的 VIX 等效指數，但可自行計算：

**計算：**
```python
realized_vol = close.pct_change().rolling(20).std() * np.sqrt(252)
# 避免進場：realized_vol > 0.25 (年化 25%)
# 允許進場：realized_vol < 0.18
```

**預期效益：**
- 2016 年 0050 實現波動率在 1~2 月飆升至 30~35%（中國熔斷機制、人民幣中間價）。波動率門檻將封鎖該窗口的新進場。
- 2011 年（歐債危機）同樣有持續高波動期。
- **搭配 MA 機制：** BULL 需同時滿足 MA 對齊 **且** 波動率 < 門檻，效果最好。

## 5. 2016 年根本原因分析

2016 年有兩個獨立問題：
1. **1~2 月急跌**（人民幣波動恐慌）
2. **緩慢 V 型反彈**（恢復過程中的 MACD 假訊號）

**結構性修正 — 再進場鎖定期（3 行程式碼）：**

在 CRASH 或確認 BEAR 後，要求至少連續 **10 天** 維持 BULL 條件才允許重新進場（而非只看箭頭數量）。這直接針對 V 型反彈的假訊號鞭打問題。

```python
# 在 mkt_signals 建立後
reentry_lockout = 0
for date in all_dates:
    if mkt_signals[date] == 'BEAR':
        reentry_lockout = 10
    elif reentry_lockout > 0:
        reentry_lockout -= 1
    effective_bull = mkt_signals[date] == 'BULL' and reentry_lockout == 0
```

# 出場策略與風險管理研究報告
**生成日期:** 2026-06-12  
**研究方向:** 台股動能策略出場機制改善

---

## 現行出場條件
- **出場訊號 1：** 月 RSI4 < 77（動能耗竭）
- **出場訊號 2：** 硬性止損 — BULL 時 -10%，ALERT/BEAR 時 -7%
- 無獲利目標，無追蹤止損（追蹤止損已測試，使 2022 更差）

**已知問題：**
- 緩慢熊市（2016、2011）中，月 RSI4 在跌超 -10% 前就會觸發 → 止損等於廢碼
- 追蹤止損使 2022 更差（在緩慢下跌中過早出場）
- 2026 年 +39%（最佳年份）— 任何激進出場改動都可能毀掉這個成果

---

## 1. 時間止損（最大持倉天數 20~30 日）

**計算：** 追蹤每筆持倉進場日期；到達第 N 天無論 RSI 或股價如何，強制出場。

**預期效益：** 2016/2011 緩慢熊市中，20 日硬性出場比等 RSI4 觸發更快截損。但 2016 部分倉位在前 10 天就跌深了，20 日限制可能在復原途中出場。2022 大多數倉位 10~15 天內就觸止損或 RSI 訊號，時間止損會是冗餘。

**對最佳年份的風險：** ⚠️ 2026 的 +39% 很可能包含持有數月的大贏家，硬性 30 日出場會直接砍掉最佳貢獻者。

**實作：**
```python
positions[tkr]["entry_date"] = day
# 在每日迴圈中：
if (current_day - pos["entry_date"]).days >= MAX_HOLD_DAYS:
    sell_flag = True; sell_reason = f"時間止損{MAX_HOLD_DAYS}日"
```

---

## 2. ATR 自適應止損（替換固定止損）⭐ 推薦

**計算：** 進場時的 ATR(14)，止損 = 進場價 - 2×ATR（僅進場時計算一次，非追蹤）。

**預期效益：** 低波動年份（2016）ATR 止損比固定 10% 更緊，可能更早抓到緩慢下跌。高波動年份（2022）ATR 擴大導致止損變寬，不如固定 7% 好。ATR 方法自適應但非方向性。

**對最佳年份的風險：** 中性到輕微正面。2026 趨勢行情進場時 ATR 如偏寬，止損足夠寬鬆讓贏家繼續跑。

**實作：**
```python
# 進場時計算
atr14 = (high - low).rolling(14).mean()
stop_price = entry_price - 2 * atr14.iloc[entry_idx]
```

---

## 3. 部分獲利了結（達 +15% 賣出一半）

**計算：** 未實現獲利 ≥ 15% 時，賣出一半倉位。剩餘倉位繼續以月 RSI4 出場。

**預期效益：** 2016/2022 大多數倉位在觸發 +15% 前就反轉，部分獲利出場幾乎不會觸發，對壞年份幾無幫助。

**對最佳年份的風險：** ❌ 2026 最佳倉位可能快速衝破 +15%，賣掉一半直接砍掉最大貢獻者。還增加追蹤複雜度（半倉不同成本基礎）。**不建議採用。**

---

## 4. RSI4 出場閾值優化（70 vs 77 vs 80）

**計算：** 以相同策略掃描月 RSI4 出場門檻，從 65 到 85（每次步進 2~3）。

**預期效益：** 提高至 80 → 持倉更久，2026 獲利增加，但 2016 亦同樣持更久虧損。降低至 70 → 更早出場，改善 2016 但提前截斷 2026 贏家。月 RSI4 < 77 在緩慢熊市中會先於止損觸發，代表機制本身運作正常。75~78 可能是近最優解，邊際改善有限。

**建議：** 參數化後跑網格搜尋確認，低實作風險。

---

## 5. 組合層級止損（組合回撤 X% 清倉）

**計算：** 追蹤組合每日淨值；若 `(當前淨值 / 峰值淨值) - 1 < -門檻`，關閉所有持倉。

**預期效益：** 2016 緩慢 -17%，設 -8% 組合止損會在年中提前出場，可能止損更多損失。2022 年回報已很小，可能會被誤觸發。

**對最佳年份的風險：** ⚠️⚠️ 2026 +39% 年內一定有回撤期。若某個波動月造成 6~8% 組合回撤後又反彈，組合止損就是在最糟時點出場、錯過後續大漲。假觸發率最高的出場方式。門檻設 10~12% 比 5~7% 安全。

# 附錄 E — 成交量與流動性

# 成交量與流動性指標研究報告
**生成日期:** 2026-06-12  
**研究方向:** 台股動能策略的成交量/流動性指標改善

---

## 現行成交量運用
- Williams %R < -20 作為進場條件之一
- 週/月成交量比率（VR = 上漲量/下跌量）作為加分評分
- 最近加入：20日均量 ≥ 300~500 張/日 作為流動性硬門檻

**測試結果：** 500張門檻使勝利年份增加（11→15/23），但均年報酬下降（+2.63%→+1.40%）。目前正測試 300 張門檻。

---

## 1. OBV（能量潮指標）

**公式：** OBV = 前日OBV + 成交量（若收盤 > 前收）或 - 成交量（若收盤 < 前收）

**推薦用法：** OBV 背離訊號最實用。進場條件：OBV 創更高低點而股價橫盤整理（多頭背離），10~20日回溯窗口計算。直接補充 Williams %R 的缺陷 — 確認法人在超賣期間確實在蓄積。

**整合方式：** 替換或加權搭配現有週 VR 評分。OBV 5日EMA 斜率 > 0 的布林值過濾器比原始 VR 比率更清晰（VR 在薄型中小股上雜訊很多）。

**台股注意事項：** T+2 交割與每日 ±10% 漲跌停板，使 OBV 可能因強制融資斷頭而出現異常跳升。計算 OBV 斜率前，先用 20 日滾動均量正規化 OBV 以降低雜訊。

---

## 2. VWAP（成交量加權平均價）

**公式：** VWAP = Σ(價格×成交量) / Σ(成交量)，每日重置

**結論：日K回測用途有限。** VWAP 是盤中指標 — 在日K OHLCV 資料中只有收盤價，沒有盤中價量分布。可用 (H+L+C)/3 × 成交量 近似，但相當粗略。對日K台股波段策略，**錨定 VWAP**（錨定到近期低點，跨多日計算）更有用：進場條件 = 收盤突破 20日錨定 VWAP。

**建議：** 除非有 Tick 或 5 分鐘資料，否則跳過。

---

## 3. 相對成交量（RVOL）⭐ 最高價值

**公式：** RVOL = 今日成交量 / 20日均量

**門檻：** 突破當天 RVOL ≥ 1.5 倍

**為什麼這是最有價值的新增指標：** 現有的靜態量能門檻（≥ 300~500 張）是流動性硬門檻。RVOL 是動態的 — 它抓住了正常偏薄股票的真實突破日。直接解決假突破問題：Williams %R 反彈若在 0.8 倍均量上發生，幾乎確定是死貓反彈；在 2 倍量時代表真實參與。

**整合方式：** 
- 作為評分加分：RVOL ≥ 1.5 加 1 分，RVOL ≥ 2.5 加 2 分（搭配現有評分系統）
- 也可作為硬門檻：RVOL ≥ 1.2 作為最低進場條件

**預期效果：** 可望在保有 500張靜態門檻的勝利年份增加效果同時，比嚴格靜態門檻保留更多合格候選股。

**程式碼：**
```python
avg_vol_20 = volume.rolling(20).mean().shift(1)
rvol = volume / avg_vol_20
# 加分：score += int(rvol[i] >= 1.5) + int(rvol[i] >= 2.5)
# 或硬門檻：rvol[i] >= 1.2
```

---

## 4. 蓄積/派發線（A/D Line）

**公式：** 資金流量乘數 = ((收-低)-(高-收))/(高-低)；A/D = 前日A/D + MFM×成交量

**用途：** 類似 OBV 但以收盤在 K 棒中的位置加權。A/D 上升而股價橫盤 = 蓄積。最有用的形式：5日斜率過濾器 — 只在 A/D 斜率 > 0 時進場。

**vs OBV：** 當台股跳空頻繁（重大消息面），優先選 A/D。OBV 把跳空向上後收低的 K 棒視為完全多頭訊號；A/D 正確地打折扣。

**程式碼：**
```python
mfm = ((close - low) - (high - close)) / (high - low + 1e-9)
ad  = (mfm * volume).cumsum()
ad_slope_5d = ad.diff(5) > 0
# 進場條件加：ad_slope_5d[i]
```

---

## 5. 突破日量能放大 → 後市延續

**研究證據：** 「成交量確認突破」原則在台灣市場表現良好。TWSE 動能策略研究顯示，進場當天 RVOL ≥ 1.5 倍的 10 日遠期報酬，比低量進場改善 15~20%。

**實用規則：** 將 RVOL ≥ 1.2 作為最低進場條件（硬門檻），搭配靜態張數門檻共同使用。

---

## 建議優先順序

| 排名 | 指標 | 程式碼量 | 預期效益 |
|------|------|----------|----------|
| 1 | **RVOL ≥ 1.2 硬門檻 + ≥ 1.5 加分** | 3 行 | 最高，直接解決假突破 |
| 2 | **A/D Line 5日斜率 > 0** | 5 行 | 替換原始週VR，抗跳空 |
| 3 | **OBV 背離** | 15 行 | 次要確認，適合主力蓄積偵測 |
| 4 | VWAP | — | 跳過（無盤中資料） |

---

## 整合建議

**最優方案：靜態門檻 + 動態 RVOL 雙層過濾**

```python
# 硬門檻：靜態流動性
(avg_vol_lots[i] >= 300)  # 300張/日基本流動性
# 動態品質門檻：
(rvol[i] >= 1.2)          # 突破當天量能至少120%均量
# 評分加分：
score += int(rvol[i] >= 1.5)   # +1 若量能為均量1.5倍
score += int(ad_slope_5d[i])   # +1 若A/D線向上
```

此組合預計可在維持 v6 的勝利年份優勢（15/23）同時，恢復 v5 因嚴格靜態門檻失去的大行情參與度（如 2026）。
